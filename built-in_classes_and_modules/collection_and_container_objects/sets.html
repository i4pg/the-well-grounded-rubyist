A set is a unique collection of objects.

The objects can be anythingâ€”strings, integers, arrays, other setsâ€”but no object can occur more than once in the set.

Uniqueness is also enforced at the commonsense content level: if the set contains the string "New York", you canâ€™t add the string "New York" to it, even though the two strings may technically be different objects. The same is true of arrays with equivalent content.

NOTE Internally, sets use a hash to enforce the uniqueness of their contents. When an element is added to a set, the internal hash for that set gets a new key. Therefore, any two objects that would count as duplicates if used as hash keys can't occur together in a set

Manipulating set elements

  • #<< || #add || (#add? return nil if the set unchanged )

>> tri_state = Set.new(["New Jersey", "New York"])
=> #<Set: {"New Jersey", "New York"}>
>> tri_state << "Connecticut"
=> #<Set: {"New Jersey", "New York", "Connecticut"}>
> tri_state << "Connecticut" # Second one, Set unchanged!
=> #<Set: {"New Jersey", "New York", "Connecticut"}>

  • #delete

=> #<Set: {"New Jersey", "New York", "Connecticut", "Pennsylvania"}>

=> #<Set: {"New Jersey", "New York", "Pennsylvania"}>

SET INTERSECTION, UNION, AND DIFFERENCE

asia = Set.new(%i[china japan thailand uae ksa])
=> #<Set: {:china, :japan, :thailand, :uae, :ksa}>
middle_east = Set.new(%i[ksa uae egypt])
=> #<Set: {:ksa, :uae, :egypt}>

Each of these methods returns a new set consisting of the original set

  • #intersection, alias: #&
> middle_east & asia # middle_east.intersection(asia)
=> #<Set: {:ksa, :uae}>
  • #union, alias: #+ and #|
> middle_east + asia # middle_east.union(asia) || middle_east | asia
=> #<Set: {:ksa, :uae, :egypt, :qatar, :china, :japan, :thailand}>
  • #difference, alias: #-
> middle_east - asia # middle_east.difference(asia)
=> #<Set: {:egypt, :qatar}>
  • #^, an exclusive-or operator

take the exclusive union between a set and an enumerableâ€”that is, a set consisting of all elements that occur

> middle_east ^ asia
=> #<Set: {:china, :japan, :thailand, :egypt, :qatar}>

MERGING A COLLECTION INTO ANOTHER SET

  • #merge
> middle_east.merge(asia)
=> #<Set: {:ksa, :uae, :egypt, :qatar, :china, :japan, :thailand}>

>> europe = { france: :paris, italy: :roma }
=> { france: :paris, italy: :roma }

>> middle_east.merge(europe.keys)
=> #<Set: {:ksa, :uae, :egypt, :qatar, :china, :japan, :thailand, :france, :italy}>

Merging a hash into a set results in the addition of two-element, key/value arrays to the set
